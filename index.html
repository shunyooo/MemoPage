    HTMLとは

    Hyper Text Markup Languageの略。Webページを作る上で最も基本的な言語である。
    HTMLタグを用いて、ヘッダやタイトル、パラグラフなどの要素を構造的に記述する。
    要素の中に更に要素や文字列を記述するので、マトリョシカのような構造になる。

    DOCYTYPEとは

    <!DOCTYPE>はDTD(Document Type Definition:文書型の定義)を宣言するために先頭に記述するものである。
    このソースの場合、<!DOCTYPE html>と記述することで、HTML5で作成していることを明示的に宣言している。

    タグ(tag)とは

    通常 <b><要素名></要素名></b>と表記し、その間に更にタグや文字列を記述する。要素名は小文字でも大文字でも構わない。
    この場合<要素名>は開始タグ(start tag)、</要素名>は終了タグ(end tag)と呼ぶ。
    aはリンクを、pは段落を、liはリスト、といった意味を持つ要素を、タグを用いてページを構造的に作成することができる。 例えばこのページのタイトルは
    ...
    <html>
        <head>
            ...
            <title>HTML5とは？</title>
        </head>
    ...
    とtitleタグを指定することにより実装されている。 また、終了タグを省略できるもの(<li>:リスト,<dd>:定義説明,など)や、元々終了タグのないもの(<br>:改行,<img>:画像,など)もある。

    要素(element)とは

    タグで述べた際に用いるheadやbody,aやpそのもののことである。タグを用いて、要素の範囲を定めることでHTMLを構造的に記述することができる。
    要素と要素は包含関係にある場合が多く、その場合、含んでいる側の要素を親、含まれている側の要素を子と呼ぶ。

    属性(attribute)とは

    要素に情報を付加するもの。様々なものがあり、色や大きさを指定することができる。
    例えば、a要素(ハイパーリンクを定義)にはhrefというリンクのURLを付加することができ、 
    <a href="index.html">インデックスページ</a>
    と表記すれば、インデックスページというようにhref属性を付加したハイパーリンクが作成される。 このとき、hrefを属性名、"index.html"を属性値と呼ぶ。

    基本的な構造

    HTMLの基本的な構造は以下のようになる。

    DOCTYPE宣言を先頭に行い、その下にhtml要素を記述する。
    html要素は子にhead要素とbody要素のみ持つことができる。
    head要素にはタイトル名や文字コードの指定などWebページには直接見えない部分の情報を記述する。 例えば、このページでは 
    <head>
      <meta charset="UTF-8">
      <title>HTML5とは？</title>
    </head>
    というように記述されている。
    <meta charset="UTF-8">は文字コードをUTF-8に指定している。metaは英語で「-を超えた、-を含んだ、-の後の」などの意味を持ち、デフォルトで設定されているページ情報を後から指定する場合に用いるようである。
    <title>HTML5とは？</title>ではページタイトルを「HTMLとは？」に指定している。ここで指定されたタイトルはタブやウィンドウの枠欄などに表示される。
    body要素ではページの本体となる情報、つまりWebページでユーザーが直接目にする情報を記述する。

    CSSとは

    CSS（Cascading Style Sheets）とは、ウェブページのスタイルを指定するための言語。HTMLで作成した構造にCSSを適用させて、色やサイズやレイアウトなどの装飾を施し、見栄えを整えることができる。

    セレクタとは

    上の画像のように「HTMLのどの部分を装飾するか」を指定する部分。この場合、h1タグの内容にスタイルが適用される。
    タグ指定の他にも、クラス指定「.(class名){}」やid指定「#(id番号){}」も行える。
    また、「p a{}」というように入れ子を絞って指定することもできる。

    プロパティとは


    「どのスタイルを適用するか」を指定する部分。この場合、color(プロパティ)に＃000000(値)が入る。
    プロパティは一つだけではなく複数の指定が行える。

    idとは

    HTMLでタグにid属性を与えることができる。
    CSSで「#(id番号){}」と記述することで目的のidのみにスタイルを適用することができる。
    一度記述したidは同一ページ内で再び使用することはできない。

    classとは

    HTMLでタグにclass属性を与えることができる。
    CSSで「.(class名){}」と記述することで目的のclassのみにスタイルを適用することができる。
    idとは違い、記述したclassは同一ページ内で何度も使用することができる。

    CSSの適用の方法

    大きく分けて
  • ヘッダーに組み込む
  • タグのstyle属性に組み込む
  • 外部にCSSファイルを作り読み込む
    • の3つの方法がある。

    ヘッダーに組み込む場合 
    <html>
      <head>
        <style type="text/css">
        <!--
        body {color:red; width: 750px;}
        -->
        </style>
      </head>
      <body>
      </body>
    </html>
    というようにヘッダーに指定する。

    タグのstyle属性に組み込む場合 
    <h1 style="color:red;">タグの内容</h1>
    というようにタグのstyle属性で指定する。

    外部にCSSファイルを作り読み込む場合 
    <link rel="stylesheet" type="text/css" href="xxx.css">
    というようにlink要素をヘッダーで使用することで指定する。
    この場合、rel="stylesheet"で外部のスタイルシートを読み込むことを示し、typeでMIMEタイプの指定、hrefでファイルへのリンク(この場合は相対URL)の指定を行っている。

    DOMとは

    DOM（Document Object Model）とは、htmlやcssの要素に直接アクセスし、書き換えなどの操作を行える仕組み。「ボタンを押すと<p>タグ内の内容を変更する」などの動的な操作を行うことができる。様々な言語で実装が可能だが、基本的にはJavaScriptで実現する。
    基本的に「要素の取得」->「取得した要素にイベントを登録」という流れとなる。

    addEventListenerとは

    DOMにおいて、イベントを登録する(イベントが起こった時に関数が動くようにする)のに用いるメソッド。
    第一引数に「イベントハンドラ(イベントの種類)」、第二引数に「関数」、第三引数に「イベントの伝播形式」を記述する。 例えば、以下のように記述することができる。 
    <p id="hoge">アラートを実行します</p>

        <script>
        var elem = document.getElementById("hoge");

        elem.addEventListener("click", function(){
            alert('アラートを実行してます!');
        });
        </script>
    これを組み込むと以下のようになる。

    アラートを実行します


    この場合、document.getElementById("hoge")によって要素を取得し、その要素に対してaddEventListnerを行っている。"click"は要素がクリックされた時、無名関数function(){}内の処理を実行することを指定している。
    第三引数は省略してあるが、デフォルト値は「false」でバブリングフェーズで実行する設定となっている。(詳しくはイベントフェーズで)

    イベントハンドラには以下のようなものがある。
    click 要素がクリックされたとき
    dblclick 要素がダブルクリックされたとき
    mouseover 要素の上にマウスのカーソルが重なったとき
    mouseout 要素の上にマウスのカーソルが離れたとき
    load ページの読み込みが完了したとき

    DOMContentLoadedとは

    HTML読み込みの完了を知らせるイベント。以下のように記述する。 
    document.addEventListener("DOMContentLoaded", function(){
//HTML読み込み完了時に行う処理
}, false);
    WebブラウザはHTML解析中にscript要素を発見すると、HTML解析を中断し、JavaScriptを即座に実行する。そのため、そのscript要素以降にあるHTMLは、まだ読み込まれていないため、そのscriptからアクセスすることができない。
    これを回避するために「onload」イベントが発生した時にスクリプトを実行するという方法がある。 しかし、onloadイベントは画像やスタイルシート、サブフレームデータも含めた読み込みの完了を知らせるので、画像が大量にある場合などページの表示に時間がかかることがある。
    そのため、一般的に「script要素をbody要素の最後に置く方法」や「DOMContentLoadedイベントを用いてHTML読み込み完了のタイミングで実行させる方法」といった方法がとられる。このような時にDOMContentLordedを用いることができる。 　　

    イベントフェーズ

    DOMはその入れ子構造から、親子に同種のイベントが設定されていた場合、思わぬ挙動をとってしまうことがある。これを避けるために、制御する必要がある。

    例えば、以下のように入れ子になった要素があったとする。 
    <div id="outer">outer
  <div id="inner">inner
  </div>
</div>
    この時イベントが発生した時、そのイベントは「キャプチャフェーズ」「ターゲットフェーズ」「バブリングフェーズ」の3つのフェーズを経て伝播する。 例えば、id="inner"の要素でイベントが発生した時、以下のように伝播していく。



    これはaddEventListenerの第三引数(true,false)で用いられ、ここでは「キャプチャフェーズ」、「バブリングフェーズ」のどちらのフェーズで実行するかを指定する。 
    <script>
document.getElementById('outer').addEventListener('click',function(){alert('outer');},true);
document.getElementById('inner').addEventListener('click',function(){alert('inner');});
<script>
    と、outerの第三引数をtrueにし、innerをクリックした場合、innerで発生したイベントのキャプチャフェーズでouterが実行される。よって、outer->innerの順にアラートが表示されるだろう。

    getElementByIdとは

    要素をidによって取得するメソッド。 
    element = document.getElementById(id);
    というように記述することで、ドキュメント内で引数のid属性値を持つ要素がelementに代入される。
    この取得した要素に対し、イベントを登録するなどの操作を行うことができる。
    他にも要素のゲッターは以下のようなものがある。 
    elements =　document.getElementsByClassName(class_name);//クラス名で取得
elements = ducument.getElementsByName(name);//name値で取得
    これらはgetElementsとあるように、該当するものをリストで返してくれる。
